# Feature: README badge

## Status: AS-BUILT 2026-07-17

## 1. What this feature is for

A player can show their Token Forest on their GitHub profile with a small live image — a "badge" — that displays their tree species, growth stage, total token count, and worldwide rank, and refreshes itself as their forest grows. The `/badge` page teaches a player how to get theirs and paste it in correctly.

## 2. Happy path

### 2.1 A player gets their badge

Inside the desktop app, once the player has turned the leaderboard on, they open the menu (Leaderboard → Badge Setup), read the short guide, and click "Copy My Leaderboard Badge". The app copies one line of Markdown to their clipboard with their own ID already filled in.

### 2.2 A player puts it on GitHub

The player creates the special GitHub profile repository (named exactly after their username), pastes the copied line into its README, and saves. Their profile now shows the badge, which updates itself over time.

### 2.3 A visitor learns how on the website

The `/badge` page walks a reader through the same flow, in their own language, with screenshots of the app menu, the Badge Setup window, and an example of the finished badge. It also answers "why is my badge blank / generic".

## 3. Edge-case behavior

### 3.1 Player has not opted into the leaderboard

#### Behavior

- The badge image falls back to a generic Token Forest picture — no name, no stats.
- This is identical to what a stranger or a banned player's badge shows, so the badge never reveals whether a particular account exists or is banned.
- The page tells the reader this is the most common cause of a "blank" badge and to turn the leaderboard on first.

#### Classification

[Smoke test only]

### 3.2 The app renames a button or menu item

#### Behavior

- The website page quotes the app's button/menu labels (in all four languages). If the desktop app renames them, the page goes stale and misleads readers.
- Mitigation: the labels live in one `APP` map at the top of the page with a comment pointing at the app's source of truth. Treat an app label change as requiring a matching page edit.

#### Classification

[Smoke test only]

### 3.3 Badge looks out of date

#### Behavior

- The SVG is cached at the CDN for about 10 minutes, so a very recent score change may not show immediately. This is intended (READMEs must load fast) and the page says so.

#### Classification

[Smoke test only]

## 4. Who can use this

- Anyone can read the page. Getting a personalised badge requires the desktop app plus opting into the leaderboard.

## 5. External dependencies (plain language)

- The badge image is generated by this site from public leaderboard data.
- The instructions depend on the **desktop app's** Badge Setup feature and its exact button labels — a cross-repository coupling (Token-Forest app ↔ this website). See § 3.2.

## 6. Deferred / unresolved

- (resolved) The example badge is now the live `/badge/demo.svg`, so it always shows the current design; there is no static screenshot to keep in sync.

## 7. Out of scope

- Generating a badge from the website (impossible by design: the site's leaderboard is anonymous and does not know who "you" are; only the app holds your ID).

## 8. Decision history

- **The page quotes the real app labels per language, and shows real screenshots.** The earlier version described an invented menu label ("Copy my leaderboard badge") in English for all locales and named the wrong entry point; it would have sent players hunting for a control that does not read that way in their app. Rewritten 2026-07-17 to match the app (Leaderboard → Badge Setup → Copy My Leaderboard Badge), with the opt-in-first prerequisite made explicit. (CEO asked for a correct, illustrated, multi-language page.)
- **Screenshots live under `/badge-guide/`, not `/badge/`** — the latter is the live SVG route `app/badge/[userId]`, which would try to serve a PNG name as a badge.
- **The page is linked from the footer and listed in llms.txt / llms-full.txt** — it was previously reachable only via the sitemap.
- **Badge layout (2026-07-17): the token count is the hero.** The stats row now leads with a bold, enlarged, green token count, with species and stage trailing it in muted small text (was `species · stage · tokens`, all one muted size). Applies to every live badge, since the SVG regenerates on each README view. Both the demo and the real-user path share one `badgeInner()` helper so they can't drift.
- **The demo badge shows "Username" (not a fake person's name)** so it reads as a template of your own badge.
- **A "Show your badge" section was added to the leaderboard page**, below "How to join", with the live sample and a link to `/badge` — the highest-intent entry point (a player looking at the standings is the one most likely to want the badge). The footer link was renamed from "README badge" to "Show your badge" (4 locales).
