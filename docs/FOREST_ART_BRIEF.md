# 森林页面美术优化需求(/forest-preview 原型 → 正式美术)

> 状态:待 design 认领
> 日期:2026-07-13
> 前置阅读:`docs/WEB_FOREST_PRODUCT_VISION.md` §5(森林模型)、§8(视觉方向)
> 现状原型:`ethan` 分支,`npm run dev` → `http://localhost:3000/forest-preview`
> (点「进入森林」后可交互;右上角可切换昼/夜、拍照模式隐藏 HUD)

## 一句话任务

**整体结构已拍板,不要改布局;这次只做画面升级**——把原型里的程序化占位美术(SVG 色块/渐变)替换成与现有树木 sprite 同风格的正式像素美术。

---

## 一、锁定项(不要动)

以下都是拍过板的,优化时保持不变:

1. **单屏农场式构图**:1680×1050 舞台,整片森林一屏全见,无镜头滚动。
   田地是梯形透视:后缘 y=430(x 310→1370),前缘 y=1000(x 90→1590)。
2. **四个主题角落与树位**(所有坐标在 `app/forest-preview/forest-scene.tsx` 顶部常量表):
   - 雪角(左后):圣诞树大小各一,白雪地块
   - 沙角(右后):仙人掌大小各一 + 破篱笆,沙地地块
   - 神社角(右中):鸟居 + 樱花树大小各一 + 两盏石灯 + 竹篱 + 风铃
   - 果园角(前排 C 位):苹果树大小各一 + 秋千 + 果篮 + 宝箱
3. **遮挡与景深规则**:y 排序(越靠下越前)+ 后排缩小(0.88–1.0)。
4. **树与装饰 sprite 沿用现有资产**(`public/sprites/`,32 棵树 + 装饰),不重绘。
5. **昼夜两态**、HUD 布局、树档案卡、拍照模式等交互不变。

## 二、画面问题清单(按优先级)

| # | 问题 | 现状 | 要什么 |
|---|---|---|---|
| 1 | **田地本体** | 纯色渐变 + 斜条纹 pattern,太"数学" | 像素草地:草叶噪点、耕作行痕、明暗块;前缘台地立面(土层剖面)也要像素化 |
| 2 | **主题地块** | 雪/沙/苔是光滑椭圆色块 | 不规则像素碎边地块,与草地间有 2–3px 锯齿过渡;雪面有高光点,沙面有石粒 |
| 3 | **小径** | SVG 平滑曲线,像涂鸦 | 像素土路:石子/车辙细节、自然分叉;大门口一段做出入口感(门柱/踏石可选) |
| 4 | **植被小物** | 灌木/野花/草丛/石头是极简几何 SVG | 成套像素小物,每种 2–3 变体;四角可出主题版(雪石、沙漠草、樱瓣花) |
| 5 | **树影** | 径向渐变椭圆 | 像素椭圆阴影(两档大小/透明度) |
| 6 | **天空** | 纯渐变 + 椭圆拼的云 | 像素云 ×3 变体、更好看的太阳/月亮;夜空可加淡银河带 |
| 7 | **地平线/背景** | 田地悬浮在渐变背景上,没有"落点" | 田地四周补远景:树线剪影/草原底衬/薄雾,让地接住(昼夜各一版或可调色) |
| 8 | **围栏排布** | 前后排同一 Fence 素材重复 | 转角件 + 门柱件;后排出简化远景版(可选) |

## 三、交付物清单(建议规格)

- 坐标系 = 1680×1050 舞台;像素密度对齐现有 sprite(参考 AppleTree_8 为 759×823 的高清像素风)。
- 透明底 PNG,**锚点 = 底边中心**;放 `public/sprites/forest/`,小写连字符命名。
- 接入方式:替换 `forest-scene.tsx` 顶部常量表里的引用即可(或 design 直接改该文件,结构别动)。

## 四、风格与验收标准

1. **唯一硬标准:和现有 32 棵树同屏不违和。** 调色板直接从现有 sprite 取色,别引入新色系。
2. **昼夜都要过**:夜晚整体有 brightness(.62–.72) 滤镜,资产在暗色下仍要可读、雪地不发灰蓝脏色。
3. **拍照模式(无 HUD)截图能直接当明信片发。**
4. 新增资产总量建议 < 2MB(页面是营销站的一部分)。
5. 交付后在原型页里实际替换验收,不看单图。
